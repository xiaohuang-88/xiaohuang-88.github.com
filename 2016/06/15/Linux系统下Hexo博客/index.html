Hexo博客换计算机（从Linux换成Windows）

  • 拷贝完整的博客源码文件夹myblog

  • 在欲使用的windows计算机上，安装git和node两个依赖软件，之后安装Hexo

    • 下载地址和安装方式，见Hexo主页
    • 建议将git和Node安装在同一个文件夹下，例如，\myBlog\node和\myblpg\Git
    • 配置Node和Git运行所需的环境变量
    • 运行git-bash，在Node框架下，安装Hexo

      $ npm install -g hexo-cli

  • 将博客文件夹拷贝至Node和Git所在文件夹，即\myBlog\myblog

  • 配置当前计算机和github服务器之间进行交互传输所需的ssh key，参见如何搭建一个独立博客——简明Github Pages与Hexo教程

    • 检查计算机上现有的ssh key

      $ cd ~/.ssh

    • 根据所用邮箱地址，生成新的ssh key（使用key文件的默认存放位置）

      $ ssh-keygen -t rsa -C “邮件地址@youremail.com”
      Generating public/private rsa key pair.
      Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa，即为默认存放位置):<回车就好>

    • key文件备份，将ssh key从默认存储位置复制至Git所在文件夹中

      在Git软件所在文件夹\myblpg\Git中，新建.ssh文件夹
      在windows下，需要在cmd模式下，使用md命令创建文件名前面带有.符号的文件夹
      $ md .ssh 在当前目录下创建.ssh文件夹

    • 添加ssh key至GitHub

      • 打开本地/Users/your_user_directory/.ssh/id_rsa.pub文件（即ssh key的默认存放位置）。此文件里面内容为刚才生成的密钥。如看不到该文件，需要设置显示隐藏文件。
      • 登陆github系统。点击右上角的 Account Settings—>SSH keys —> add SSH keys
      • 在标题框中，添加这个新key的描述标签。例如，如果此时使用个人电脑处理Hexo博客，可将该key称为”Hexo-personal-computer”
      • 把本地生成的id_rsa.pub文件中内容，复制至key文本框中
      • 测试ssh连接和设置用户信息

        $ ssh -T git@github.com 测试ssh连接

Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字必须是你的真名，而不是GitHub的昵称。

$ git config –global user.name //用户名
$ git config –global user.email //填写自己的邮箱

Hexo常用命令

常用命令：

  • hexo new “postName” #新建文章
  • hexo new page “pageName” #新建页面
  • hexo generate #生成静态页面至public目录
  • hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server)
  • hexo deploy #将.deploy目录部署到GitHub

常用复合命令：

  • hexo deploy -g
  • hexo server -g

简写：

  • hexo n == hexo new
  • hexo g == hexo generate
  • hexo s == hexo server
  • hexo d == hexo deploy
